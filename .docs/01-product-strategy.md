# Fase 1. Estrategia de Producto

## Objetivo

Construir un portfolio que funcione como producto editorial y demostración técnica.

## Audiencia prioritaria

- CTO
- VP Engineering
- Engineering Manager
- Recruiter técnico
- Cliente potencial

## Posicionamiento

La propuesta no es "soy desarrollador". La propuesta es:

- Diseño productos.
- Modelo dominios.
- Construyo arquitecturas.
- Escalo plataformas.
- Integro IA con criterio técnico.
- Opero sistemas listos para producción.

## Mensaje central

> Esta persona puede diseñar, construir, desplegar y evolucionar productos complejos de extremo a extremo.

## Narrativa del sitio

1. Software ecosystems.
2. Arquitectura de software como ventaja competitiva.
3. Casos reales de SaaS, IA y operaciones.
4. Procesos, trade-offs y decisiones técnicas.
5. Capacidad para ejecutar en producción.

## Priorización de contenido

### Alta prioridad

- Home
- Projects
- Experience
- Architecture
- Contact

### Media prioridad

- Blog
- Skills matrix

### Baja prioridad

- Integraciones externas avanzadas
- Laboratorios interactivos complejos

## CTA principal

Contactar para conversación técnica o de contratación.

## No objetivos iniciales

- Marketplace
- Auth compleja
- Panel admin
- CMS completo desde el día 1
- Multi-tenant real en el portfolio
